XBN-Java: Release notes

Current release: Version 0.1.5.1

Build logs, permalink


New code:
  • com.github.xbn.io.PlainTextFileUtil#openWriteClose

Fixed:
  • com.github.xbn.util.copyval.AbstractValueCopier#getValueCopy Required explicit cast to be future-compatible with Java 8.

Build:
  • Changed jar file creation to not delete old jars. Instead, they are updated. This prevents Windows from randomly locking these files, and therefore randomly crashing the builds, because they can't be deleted. (They can be manually deleted, but not automatically. See http://stackoverflow.com/questions/77436/ant-is-not-able-to-delete-some-files-on-windows

Code:

  • Upgraded Template Featherweight to 0.1.1.3
  • Upgraded Codelet to 0.1.4.2

Version 0.1.5

Build logs, permalink

Docs: Lots of fixed links.


New code:
  • Package: com.github.xbn.util.matrix
  • Boolean constructor in com.github.xbn.number.NumberBound deprecated in favor of boolean enum version.
  • com.github.xbn.number.NumberUtil.getRandomIntBetweenInclusive (replacement for now-deprecated getRandomBetweenInclusive), isPrime
  • com.github.xbn.number.NumberInRange.getIntersection, getOverlap, doesOverlap, getMinGivenIncl, getMaxGivenIncl

Backend:

  • XBN-Java is now compatible with Eclipse, which may be used for all XBN-Java project development, or just for debugging. Class files are now output to a "bin" directory (located in the sandbox root directory), and the system's classpath environment variable is entirely ignored.
  • Upgraded Template Feather to 0.1.1.2: http://templatefeather.aliteralmind.com
  • Upgraded Codelet from 0.1.4.1: http://codelet.aliteralmind.com
  • Starting with version 0.1.4.1, all historical versions are entirely kept on aliteralmind.com. The current version is always available any any of these urls (1 redirects to 2, redirects to 3):
    1. http://xbnjava.aliteralmind.com
    2. http://aliteralmind.com/docs/computer/programming/xbnjava/current
    3. http://aliteralmind.com/docs/computer/programming/xbnjava/xbnjava-0.1.5/documentation/javadoc/overview-summary.html
  • Added optional link checker to build (which must-be-called-manually).
