XBN-Java: A collection of generically-useful backend (non-GUI) programming utilities, featuring shareable self-returning method chains, {@code FilteredLineIterator}, and regular expression convenience classes--XBN-Java is the foundation of Codelet.

  1. XBN-Java is hosted by GitHub: Project page, issue tracker
  2. JavaDoc: This installation, latest version, download zip, generation report
  3. Download source, including build files

  Installation

XBN-Java requires Java 1.7 (it may be compatible with earlier versions).

Download the jar files for XBN-Java and all the jars it depends on.

Adding the following elements to your classpath will install XBN-Java:

  1. {@code INSTALLATION_DIR\xbnjava_0p1p0_and_all_dependency_jars\commons-collections4-4.0.jar}
  2. {@code INSTALLATION_DIR\xbnjava_0p1p0_and_all_dependency_jars\commons-io-2.4.jar}
  3. {@code INSTALLATION_DIR\xbnjava_0p1p0_and_all_dependency_jars\commons-lang3-3.3.2.jar}
  4. {@code INSTALLATION_DIR\xbnjava_0p1p0_and_all_dependency_jars\guava-16.0.1.jar}
  5. {@code INSTALLATION_DIR\xbnjava_0p1p0_and_all_dependency_jars\xbnjava_0p1p0_all.jar}

Details and alternatives:

  1. XBN-Java: Choose exactly one of the following (the above classpath-string assumes you've chosen {@code xbnjava_0p1p0_all.jar}):
    1. {@code xbnjava_0p1p0_all.jar}: Core classes, example code, and unit tests
    2. {@code xbnjava_0p1p0_core.jar}: Core classes only
  2. External libraries:
    1. Apache Commons IO 2.4
    2. Apache Commons Lang3 3.3.2
    3. Less critically: Commons Collections 4 (needed for {@code com.github.xbn.experimental.}{@code listify}, and some {@code com.github.xbn.}{@code array}, mostly in {@code com.github.xbn.}{@code helper})
  3. Google Guava, version 16.0.1

Building XBN-Java

In addition to the above, building XBN-Java requires the following:

  1. Apache Ant 1.9.3
  2. JUnit 4.8.2
  3. Codelet 0.1.0 (installation instructions). Codelet is only necessary for displaying example code in the JavaDoc. JavaDoc will still work, but example code will be omitted, and one warning will be displayed for every non-working taglet. To install Codelet, add these additional elements to your classpath:
    1. {@code INSTALLATION_DIR\xbnjava\z_build\jar_dependencies\templatefeather_0p1p0_all.jar}
    2. {@code INSTALLATION_DIR\xbnjava\z_build\jar_dependencies\codelet_0p1p0_core.jar}
    and install its taglets

Open a command prompt to the XBN-Java root installation directory, and call "{@code ant help}" for more information. The build process already has the Codelet taglets installed.
